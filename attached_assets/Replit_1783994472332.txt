Please refactor the Mary English unlock/save/restore system.

This is NOT a Review Reward-only fix.

The goal is to redesign how unlockable content is stored and restored.

The following four systems are included in this change:

- Outfits
- Backgrounds
- Review Rewards
- Heart Gauge

Please do NOT modify the existing behavior of:

- Daily Talk 10-rally popup
- Weekly 7-day streak popup

They are already working correctly.

--------------------------------------------------
1. Core Design
--------------------------------------------------

Separate the following two concepts in the JSON.

1. Current State
2. Popup / Presentation for this session

The application must restore the current game state directly from the imported JSON.

Do NOT infer unlocks from:

- Heart changes
- Level differences
- Review Challenge completion
- Any other gameplay events

The imported JSON is the source of truth.

Popup flags are presentation only.

Showing a popup must NEVER modify:

- XP
- Level
- Hearts
- Task counts
- Unlock states
- Selected wardrobe items

--------------------------------------------------
2. JSON Structure
--------------------------------------------------

Please store the new fields directly under the existing "progress" object whenever possible.

Example:

"progress": {

  "outfitUnlockedLevel": 1,
  "backgroundUnlockedLevel": 0,
  "reviewRewardUnlockedCount": 1,
  "hearts": 1,

  "showOutfitPopup": false,
  "showBackgroundPopup": false,
  "showReviewRewardPopup": true,
  "showHeartPopup": true

}
